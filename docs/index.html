Communicate your design intent to an LLM.

Vibeliner is a native macOS menu bar app that lets you capture, annotate, and package screenshots for AI coding tools.

See a bug? Capture it → annotate it → paste directly into Claude, ChatGPT, or Cursor.

The annotation editor — pins, arrows, and notes on your screenshot

the problem

You see a visual bug in your app. The padding is off, the border radius is wrong, a card is misaligned. You know exactly what's wrong — but describing it to an LLM in text is painful.

"The second card in the third row has too much padding on the left side, and the border radius doesn't match the design spec" — by the time you've typed that, you've lost momentum. And the LLM still might fix the wrong thing.

Vibeliner replaces that paragraph with a screenshot, a numbered pin, and two words: "padding wrong."

what the LLM sees

Every capture produces two things: an annotated screenshot with numbered marks baked in, and a text prompt listing each annotation with its note. The LLM gets both — the visual context and the structured instructions.

This is a screenshot of my running app. Numbered pins point to specific issues and arrows point at or between elements. Each annotation has a number and a description. Fix each issue: 1 [pin] padding too tight 2 [arrow] wrong border radius 3 [rectangle] this card needs more height Make the changes and verify they match the design.

The prompt is fully customizable — edit the preamble, tool descriptions, and footer in Settings. The tool description adapts automatically based on which annotation tools you used.

two modes

Vibeliner works with both terminal-based AI tools and chat-based AI tools.

IDE mode

For Claude Code, Codex, Cursor terminal. One paste — the prompt includes the file path to the screenshot on disk. The LLM reads both.

App mode

For Claude.ai, ChatGPT, web tools. Two pastes — copy the prompt text, then copy the image separately. Both buttons are always visible.

multi-image composites

Capture multiple screenshots and Vibeliner stitches them into a single annotated filmstrip. Label each image with a role — Observed, Expected, or Reference — so the LLM knows what it's looking at versus what it should look like.

Annotations span across images. Draw an arrow from the current state to the design spec. The prompt auto-prefixes each note with the image name so there's no ambiguity.
