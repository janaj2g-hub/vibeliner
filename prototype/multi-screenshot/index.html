Prototype Exploration · Multi-screenshot capture

Capture the broken UI and the target UI in one pass.

This prototype explores a first-class before/after workflow for Vibeliner. The core idea is a dedicated compare mode that captures two screenshots in sequence, opens one shared annotation editor, and exports a prompt that clearly separates observed vs expected context.

Recommendation

Approach A is the clearest entry point: explicit compare mode, two capture moments, zero ambiguity about image roles, and the least confusing mental model.

What We’re Stress-testing

Capture pacing, split editor readability, shared annotation numbering, prompt labeling, and whether the divider + two-pane layout feels like enough structure without becoming heavy.

Why A feels strongest

It makes “compare mode” explicit before anything is captured. That removes ambiguity from naming, autosave, prompt wording, and the mental model in the editor.

Key UX bets
  • The second capture prompt needs to feel guided, not repetitive.
  • Shared numbering across both canvases helps the prompt read as one narrative.
  • The split editor needs labels that stay visible even when the images look similar.
Prototype implications
  • New compare-mode entry point in the popover or via modifier key.
  • Prompt generation and autosave become image-role aware.
  • Undo/redo and annotation storage likely need a shared session model with per-image coordinates.

This prototype is intentionally HTML-only so we can validate the interaction model before pushing the AppKit editor toward a more invasive architecture change.
